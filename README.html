为什么会降

三个 agent 都把“参数很多 + 步数很少”写成较强惩罚，并把高学习率的过拟合/不稳定风险提前编码。这个先验听起来合理，却在本题集里系统性低估了容量收益；尤其一维回归中，大模型经常仍占优。25 项公式修订只改变了 5 个答案，三次合计从 61 个初选正确降到 59 个终选正确。

新增 prompt 的核心约束

For each family, write:
Score_family(candidate | question)
  = weighted benefits - weighted risks + interaction terms

At minimum include exact/estimated trainable parameters, optimizer steps,
non-monotonic learning-rate fit, optimizer suitability, architecture,
normalization/residual interactions, regularization, underfit and overfit risk.

Pass 1: score every option and choose.
Audit: enforce repeated-candidate consistency and detect double counting.
Revision: change only shared family-level weights, without labels; re-score all
affected questions. Save every contribution, risk flag, revision, and answer.

完整 prompt：artifacts/quiz_attempt_60/gpt56_sol/heuristic_formula/prompt.md。三份完整 60+60 轨迹、评分和对照统计位于同目录。

下一步

公式方向仍值得保留，但权重不能凭直觉在测试集上继续调。应使用独立 calibration split 学习参数量、学习率、优化器及交互项的权重，冻结公式后再评估 held-out 题集。

主要原因

5.6-SOL 盲答时经常形成“短预算下高学习率/快速优化器更占优、容量过大更危险”的共同启发式。这个先验在一些题上合理，但在本题集里会系统性错过容量收益，尤其是一维回归：单题盲答按 family 看，bigram_lm 60.0%，multivariate_regression 45.0%，univariate_regression 只有 20.0%；整套盲答的一维回归也只有约 24.2%。

共同偏差会让投票失效

单题 fresh-agent 盲答是 25/60 = 41.7%，整套盲答均值反而是 23.67/60 = 39.4%；这说明对 5.6-SOL 来说，一次性看到全套题没有产生 5.5 那种横向校准收益。整套盲答 6 个 agent 的多数投票也只有 24/60 = 40.0%，因为很多错题上高度一致地选同一个错误启发式。

反馈后能明显恢复

同一个模型在顺序反馈里升到 72.2%，说明它能从真实反馈里修正规则。更合理的解释是：无标签盲答时默认先验不适配这个 synthetic generator；不是模型完全不能学，也不是上下文压缩本身导致崩掉。

和 5.5 的比较要带协议注脚

5.6-SOL 的单题盲答已经补跑，协议是每题 fresh subagent；全程反馈仍是分段 continuation，不完全等价于 5.5 的 CLI 连续记录。它可以作为行为观察，但不应被写成严格模型排行榜。

一句话汇报

5.6-SOL 在无反馈盲答中偏低，单题 41.7%、整套 39.4%，主要是共同启发式先验在本题族上错得更一致；一旦给顺序反馈，正确率提升到 72.2%，说明低分主要来自 blind prior mismatch，而不是反馈学习能力缺失。

Behavioral interpretation

为什么不同上下文会改变正确率

这更像行为学测量，而不是一个静态 leaderboard。
单题盲答
只能调用一般先验
模型只能根据 optimizer、学习率、模型容量等常识猜测；无法知道本题集生成器的具体偏好。
整套盲答
可以做横向比较
同一候选、相似数据族、重复配置会在多题中出现；模型能用题间一致性校准自己的启发式。
顺序做10题
开始形成经验规则
模型只能在当前 10 题内积累 lesson；小组之间 reset，所以它能学习，但记忆长度受限。
顺序做全部题目
最接近人类刷题视角
一个 agent 从头做到尾，前面题目的反馈都可以进入后续推理；记录器强制版只是在工具层保证先答后看。
长上下文
有收益，也有衰减风险
整套接龙会变长；但 subagent 具备自动上下文管理，主要风险不是上下文压缩 bug，而是协议审计不够硬。
Recommended wording

明天汇报建议

用这几句话压住争议点。
01主结论clean
在 60 道三选题主口径上，GPT-5.5 high 单题盲答约 46.7%，整套盲答约 57.8%，顺序做10题约 65.0%，顺序做全部题目约 79.4%。
02上下文解释behavior
正确率随可用上下文增加而上升，说明模型并非只在做单题静态推理，而是在利用题间结构和反馈经验。
03协议解释protocol
正式结论要同时报告 subagent 可见上下文、禁止项、输出 JSON schema、模型 effort 和评分方式。
Take-home

这组实验最有价值的不是单个分数，而是展示 GPT 的“解题行为”会被上下文设置显著调制： 从单题先验、到整套比较、再到反馈学习，表现呈现可解释的阶梯式变化。
