## P0：题库有效性与发布筛选

### R1. 防止单一启发式泄漏答案

题库不应让“总选参数量较大”或“有 KAN 就选 KAN”成为高命中率规则。对每个候选对和题包计算至少以下 rule baseline：

1. 选择 trainable parameter count 更大的 candidate；
2. KAN vs 非 KAN 时一律选择 KAN；
3. 可扩展为特定 optimizer、深度或宽度的单规则。

新生成题包必须导出按 track、family、difficulty 和总体的命中率。难题子集应将每条已登记的单规则控制在接近随机水平（建议先以 40%–60% 为工作目标，并设置最小样本量）；普通题包可保留有教学价值的规律题，但不能让某条规则统治整包。

证据：`q_041099`、`q_4420e1`、`q_9ec784`。这是一项**题包级**约束，不应通过篡改单题答案实现。

### R2. 参数量差距与比较公平性

candidate 的 `trainable_parameter_count` 已显示，但未来题目生成和审计需要同时记录：两侧参数量、比值、对数比值、是否触发 warning/fail，以及例外原因。架构/混合题不应常规地产生数十倍差距的候选对。

建议采用“profile / track 级门槛 + 明确 diagnostic 例外”，而不是声称一个跨任务通用的固定数字：

- normal target：参数量比尽量不超过一个待定的目标范围；
- warning：显著不平衡但仍可人工查看；
- fail：默认不入题；
- 例外：专门研究容量或计算预算时，必须在题面和 provenance 标为 diagnostic。

先用 `q_6b7b54`、`q_64454b`、`q_b9731f` 的实际 spec 回填分布后，再把阈值写入新 profile；冻结的 v1/v2/v2.1/v2.2 不回写。

### R3. KAN track 的胜率与难度平衡

`classification_kan_v2.2_expanded` 不能演变为“几乎总是 KAN 胜”的演示。新 KAN profile / question run 需要：

- 同时保留 KAN 胜、MLP 胜、容量相近但因过拟合/优化动态而反转的对照；
- 对一层 vs 两层这种直接容量比较标为 easy，并限制其在难题集中的占比；
- 将 `q_41ab08`（过拟合）与 `q_70ebab`（参数量反直觉）作为正向模式，而非只按模型名称平衡。

证据：`q_5d20d3`、`q_229b84`、`q_41ab08`、`q_70ebab`、`q_4420e1`、`q_9ec784`。

### R4. 训练可判读性与题目质量门

- 对 `q_780edb` 所反映的“256 steps 后仍大多未收敛”，审计时需区分“刻意考察早期训练动态”与“预算不足导致的随机/未收敛”。前者应明确标注，后者应提高预算或筛除。
- 轨迹几乎重合的候选（`q_6fed57`）不应仅依赖图形可读性通过；需要以 effect/gap、置信信息和可辨识性共同决定是否入题。
- 不确定区间须来自重复 seed 的真实结果，不能由前端凭空推断。

## P1：题面、可视化与审计体验

### R5. 数学公式成为题面的一等内容

回归表达式、数据生成机制、损失或关键定义应支持严谨的 LaTex/MathJax 渲染，并在静态导出和本地审计界面中一致可见。目标是补充可检验的定义，而不是把整段生成代码塞进题面。

证据：`q_64454b`、`q_a0d3dd`。

### R6. 分类示意图：先核验语义，再改善呈现

`q_80adae`、`q_903d2d` 指出“红色 X 并未主要落在红色背景”。这不能直接假定为绘图 bug：如果背景表示训练集局部经验 `P(class 1)`，而 X 是带噪声的独立测试样本，单个 X 落在异色区域可能是预期现象。

需完成的工作：

1. 从 exporter / 绘图实现验证背景、训练点、测试点各自的统计含义；
2. 在图例和图注中显式写出该含义、颜色映射与“异色测试点是否可能”；
3. 若实际实现的语义与图例不一致，修复颜色或数据源；若语义正确但易误解，采用更接近 Inspector 的图形语言，或提供局部类别密度 / 决策边界的可选图层；
4. 增加渲染级回归测试，检查红/蓝映射、train/test marker 区分和 legend 文案。

这里的验收标准是“读者能正确解释图”，不是强制每个 X 都落进同色背景。

### R7. 曲线可读性、范围与不确定度

审计界面应支持选择或缩放训练 step / y 轴范围，使异常值、早期动态和几乎重合的曲线可检查；同时显示定义清楚的 uncertainty band 及其缺失态。

已约定的默认统计显示：

| 视图 | 中心线 | 阴影 |
|---|---|---|
| Linear Y | mean | mean ± std |
| Log Y（默认） | median | 10–90% 分位数 |

证据：`q_c12422`、`q_0b2cde`、`q_6fed57`。如果当前 `curves.npz` 不具有逐 seed / 逐评估点的数据，应先扩展结果协议，再启用带状区间。

### R8. 审计状态与 provenance

审计界面/导出应显示：题号/总数、track、profile、来源 run、当前反馈是否已提交、未审题列表。保存时区分：已提交、草稿、显式跳过。题目级评论和题包级评论（如 `q_9ec784` 的规则泄漏意见）应分开保存。

当前 session 的 44/46 覆盖和 4 条未提交，说明这不是纯粹的便利功能，而是防止“以为审完了”的完整性门。
